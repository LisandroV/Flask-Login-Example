Servidor web
Es un programa que gestiona cualquier aplicación en el lado del servidor realizando conexiones bidireccionales y/o
unidireccionales y síncronas o asíncronas con el cliente generando una respuesta en cualquier lenguaje o aplicación
en el lado del cliente. Generalmente se utiliza el protocolo HTTP para estas comunicaciones.

En aracne project usamos Apache el cual no es un servidor físico, sino un software que se ejecuta en un servidor.
El servidor y el cliente se comunican a través del protocolo HTTP y Apache es responsable de garantizar una
comunicación fluida y segura entre las dos máquinas. El servidor y el cliente se comunican a través del

Servidor de aplicación
Proporciona servicios que soportan la ejecución y disponibilidad de las aplicaciones desplegadas.
permite mejorar tres aspectos fundamentales en una aplicación: la alta disponibilidad, la escalabilidad y el mantenimiento.
Un ejemplo común del uso de servidores de aplicación (y de sus componentes) son los portales de Internet, dichos portales
permiten tener acceso a información y servicios (como servicios Web) de manera segura y transparente, desde cualquier dispositivo.



Servidor de correo electrónico
Permite el intercambio de mensajes de correo electrónico, ya sea entre usuarios, Servidores, cliente y servidor.
Parar llevar acabo el funcionamiento del servidor de correo son necesarios algunos protocolos:

SMTP, Simple Mail Transfer Protocol: se utiliza para el envío de correo ya sea desde un servidor de
correo a otro, o bien, desde un cliente de correo electrónico al servidor.

POP, Post Office Protocol: Se utiliza para obtener los mensajes guardados en el servidor y pasárselos al usuario.

POP3, permite acceder al buzón de correo de un usuario y gestionar mensajes.



Servidor de bases de datos
Permiten organizar datos en una o más tablas relacionadas. Todos los sistemas gestores analizados
cuentan con múltiples configuraciones de protocolos, adaptándose a los protocolos existentes y
estandarizados de la actualidad: TCP/IP, IPX, Banyan, ect; es importante no sólo el canal de
comunicaciones que está disponible para los servidores de datos sino también cómo es transmitida la información.
